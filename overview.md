# Multi-Team Sprint View

**See all your sprints in one place.**

Platform team members often work across multiple Azure DevOps teams. Each team's board only shows its own sprint — making it hard to get the full picture.

**Multi-Team Sprint View** solves this by providing a single, unified dashboard that shows:

- 🏃 **All sprints** you're involved in across every team
- 📅 **Sprint dates** with visual timeline indicators
- ✅ **Work item status** — PBIs, Tasks, Bugs with state badges
- 🔗 **Dependencies** — see Related, Predecessor, and Successor links
- 📊 **Summary stats** — total items, completion %, team breakdown

## Features

- **Cross-team visibility** — no more board-hopping
- **Smart filtering** — filter by team, sprint timeframe, work item type, or state
- **User-centric** — defaults to showing only your assigned work
- **Professional dark theme** — easy on the eyes during long planning sessions

## Getting Started

1. Install the extension in your Azure DevOps organization
2. Navigate to **Boards → Multi-Team Sprints**
3. All your cross-team sprints load automatically
